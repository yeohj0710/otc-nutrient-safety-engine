# 품질 관문과 합격 기준

## Gate 0. 연구 정체성·저장소 정합성

필수:

- 학생명 여형준, 학번 2020194025, 연구 제목, 저장소·배포 프로젝트 일치
- Kwon 전용 문자열·데이터·규칙이 여형준 분석에 포함되지 않음
- 기존 자료는 `legacy_untrusted`로 격리

실패 시 다른 작업을 진행하지 않는다.

## Gate 1. 프로토콜 동결

- 연구 유형, 5개 노드, 포함·제외 기준, 주 결과, AI 평가, 통계계획 확정
- 지도교수/검토자 확인 기록
- 프로토콜 커밋·SHA-256·등록 URL
- 결과를 보기 전 변경과 이후 변경을 구분하는 로그

## Gate 2. 검색 재현성

- 최소 MEDLINE, Embase, CENTRAL, 인용색인 1개 실행 또는 접근 불가의 공식 대체 결정
- 모든 검색식·플랫폼·날짜·hit/export/import 수·원시파일·해시
- 상위 N 절단 없음
- seed 문헌 회수 확인
- 검색전략 검토 기록

## Gate 3. 선별 무결성

- 전 기록 판정 완료
- calibration과 합의도 보고
- AI 자동 제외 0건
- 전문 제외 사유 완전
- study family 연결
- PRISMA 수치 자동 생성·산술 검증

## Gate 4. 원문 추출·근거평가

- 모든 포함 연구에 검증된 추출 레코드
- 수치·단위·locator 누락 없음
- 설계별 비뚤림 위험 평가
- 중요 결과별 GRADE
- 사례·공공자료·임상연구 역할 분리

## Gate 5. AI 평가

- 개발셋과 held-out 평가셋 분리
- 모델·프롬프트·매개변수·원시 출력 보존
- 분류·추출 성능과 95% CI
- 오류 유형 분석
- 평가셋 누출 검사
- 성능 미달 시 사용 범위 축소

## Gate 6. 근거 동결

- 5개 노드 근거 지도 완료
- 집중 노드 합성 완료 또는 미수행의 사전 기준 근거
- 모든 결과 표·그림 스크립트 생성
- `evidence-freeze-v1` 태그

## Gate 7. 규칙·소프트웨어 추적성

- 모든 released 규칙에 evidence ID, source quote, locator, certainty, reviewer
- 규칙·출처 무결성 테스트 통과
- no-reviewed-rule과 safe를 구분
- 범위 밖 기능이 논문 분석에 포함되지 않음

## Gate 8. 독립 검증

- 구현과 분리된 골드 시나리오
- 양성·음성·경계·누락·충돌·범위 밖 포함
- 검토자 합의와 전문가 평가
- 주요 지표·신뢰구간·오류분석
- sentinel critical false negative 0 또는 실패를 명시하고 출시 보류

## Gate 9. 논문 무결성

- 방법이 실제 수행과 일치
- 모든 수치가 metrics manifest와 일치
- 모든 표·그림 본문 인용
- 주장 장부 100% 연결
- 참고문헌 본문·목록 양방향 검사
- AI 생성 흔적이 아니라 연구자의 데이터와 판단 중심 문장
- 학교 형식과 PDF 렌더링 검수

## Gate 10. 최종 릴리스

- 코드 테스트 통과
- 연구 데이터·분석·논문 버전 태그
- 재현 명령 검증
- 공개 가능한 파일과 비공개 원문 분리
- README에 연구 범위·한계·AI 사용 내역
- 배포 앱과 논문 수치 일치

## 금지된 완료 표시

다음이 하나라도 있으면 `완료`라고 쓰지 않는다.

- `TBD`, `추후`, `후속 확인`, `미수행`이 핵심 결과에 남음
- 전문 미검토 후보를 근거로 사용
- 수동 입력한 PRISMA·성능 수치
- 출처 locator 없는 임상 문장
- 개발 시나리오로만 검증
- 다른 학생·다른 저장소의 수치 혼입
- CI 없는 주요 성능 지표
