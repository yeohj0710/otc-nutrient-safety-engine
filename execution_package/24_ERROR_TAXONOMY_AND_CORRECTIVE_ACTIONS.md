# 오류 분류와 시정조치

## 1. 목적

성능 수치만 제시하면 시스템이 왜 실패하는지 알 수 없다. 모든 AI·추출·규칙·시나리오 오류를 사전 분류해 빈도, 심각도, 원인, 수정 후 재발 여부를 보고한다.

## 2. 문헌 검색·선별 오류

| 코드 | 오류 | 예시 | 시정조치 |
|---|---|---|---|
| S-01 | 성분 거짓 일치 | `EPA`가 다른 약어·문맥으로 등장 | 단어 경계·문맥 확인·동의어 사전 수정 |
| S-02 | 병용 맥락 누락 | 오메가-3 연구지만 항응고·항혈전 맥락 없음 | population/medication 조건 분리 |
| S-03 | 보충제와 식이 혼동 | 식이 칼슘을 보충제로 분류 | exposure form 필수 추출 |
| S-04 | 안전성 결과 부재 | 효능만 보고한 연구 포함 | outcome 기준 강화 |
| S-05 | study family 누출 | 같은 연구의 2개 보고서가 dev/test에 분리 | group split 재생성 |
| S-06 | top-N 절단 | hit는 크지만 일부만 반입 | 전량 export·분할 로그·Gate 2 실패 |

## 3. 원문 추출 오류

| 코드 | 오류 | 판정 기준 | 심각도 |
|---|---|---|---|
| E-01 | 수치 생성 | 원문에 없는 값 | 치명적 |
| E-02 | 분모 오류 | 사건 수는 맞지만 군 분모가 틀림 | 높음 |
| E-03 | 단위 오류 | mg/µg/IU, 일일/주간 혼동 | 치명적 |
| E-04 | 시점 오류 | 1년 결과를 전체 추적 결과로 사용 | 높음 |
| E-05 | 군 방향 반전 | 노출·대조군 뒤바뀜 | 치명적 |
| E-06 | 조정치 혼동 | 조정/비조정 효과치 혼합 | 높음 |
| E-07 | locator 불일치 | 인용 위치가 값을 지지하지 않음 | 높음 |
| E-08 | not reported 오판 | 미보고 값을 0 또는 없음으로 처리 | 높음 |

E-01, E-03, E-05는 즉시 해당 AI 출력 배치를 격리하고 같은 조건의 전체 추출을 재감사한다.

## 4. 근거 합성 오류

- Y-01: 임상적으로 다른 노출을 무리하게 통합
- Y-02: 중복 모집단을 별도 연구로 계산
- Y-03: 결과 정의가 다른 사건을 합산
- Y-04: 관찰연구와 RCT를 한 효과치로 합산
- Y-05: 통계적 비유의성을 무효과로 단정
- Y-06: 낮은 확실성을 강한 권고로 변환
- Y-07: 공공 기준과 임상 효과 추정치를 같은 근거 층으로 취급

시정조치는 분석집단 재정의, 분리 합성, SWiM 전환, GRADE 재검토 중 하나 이상을 포함한다.

## 5. 규칙 오류

| 코드 | 오류 | 안전상 의미 |
|---|---|---|
| R-01 | 대상자 과일반화 | 특정 하위집단 신호를 모든 사용자에게 확대 |
| R-02 | 용량 임계값 창작 | 근거에 없는 숫자 경계 생성 |
| R-03 | 금기 과장 | 모니터링 수준을 중단/회피로 표현 |
| R-04 | 불확실성 누락 | 낮은 확실성을 확정 문장으로 표시 |
| R-05 | 근거 불일치 | 규칙과 다른 결과·대상 문헌 연결 |
| R-06 | 오래된 규범 | 개정된 기준을 계속 사용 |
| R-07 | 충돌 미처리 | 두 규칙이 상반된 행동을 반환 |
| R-08 | 미검토 규칙 노출 | draft 상태가 사용자 화면에 표시 |

R-02, R-05, R-08은 릴리스 차단 오류다.

## 6. 시스템·시나리오 오류

- V-01: critical false negative
- V-02: false reassurance (`규칙 없음`을 안전으로 표현)
- V-03: 필요한 추가 질문 누락
- V-04: 단위 변환 실패
- V-05: 자연어 정규화 실패
- V-06: out-of-scope 입력에 임상 판단 반환
- V-07: provenance 누락
- V-08: stale/deprecated rule 반환
- V-09: 개발 예시 과적합
- V-10: UI가 엔진 출력을 더 강한 표현으로 변경

각 오류는 `scenario_id`, 입력, 기대 결과, 실제 결과, root cause, fix commit, 재시험 결과를 기록한다.

## 7. 심각도

- **Critical:** 사용자가 치료 변경 또는 위험한 거짓 안심을 받을 수 있음
- **High:** 중요한 임상 조건·용량·출처가 잘못됨
- **Moderate:** 행동 등급은 맞지만 설명·질문·세부가 부정확
- **Low:** 표현·정렬·표시 문제로 임상 의미는 유지

Critical 오류는 1건이라도 최종 릴리스를 차단한다. High 오류는 수정 후 해당 클래스 전체를 재시험한다.

## 8. 논문 보고

최소 다음을 제시한다.

- 오류 분류별 건수와 분모
- Critical/High 사례의 구체적 사례표
- AI 조건별 오류 분포
- 수정 전·후 성능
- 남은 오류와 적용 범위 제한
- 사람이 개입해야 안전했던 지점

## 9. 실행환경·재현성 오류

| 코드 | 오류 | 판정 기준 | 시정조치 |
|---|---|---|---|
| X-01 | 의존성 환경 불완전 | 문서화된 테스트가 잠금 환경에서 import 또는 실행되지 않음 | 별도 연구환경과 lockfile 생성, 깨끗한 환경 재시험 |
| X-02 | 비결정적 생성물 | 같은 입력으로 재실행했는데 시각 등 비의미 필드 때문에 tracked 산출물이 바뀜 | 생성 시각을 manifest 입력으로 고정하거나 비교 대상에서 분리 |
| X-03 | 외부 설정 누락 | 필수 환경변수·DB 접근·플랫폼 정보가 없어 실행 불가 | 비밀값 없이 필요한 키·입력 형식·차단 범위를 사람 작업 문서에 기록 |
| X-04 | source adapter 필드 손실 | 원시 export에 있는 필드가 정규화 레코드에서 비어 있음 | 라이브러리 정규화 키를 확인하고 원시 태그 fallback 회귀 테스트 |
| X-05 | 정체성 marker 오탐 | 출처 경로·감사 로그의 이름이 활성 연구 내용 오염으로 잘못 판정됨 | provenance·governance 경로를 좁게 예외 처리하고 활성 데이터 오염 회귀 테스트 유지 |

X-01과 X-02가 남아 있으면 재현성 관문을 통과하지 못한다. 수정 전 출력과 수정 후 재시험 결과를 `research_v2/audit/error_log.csv`에 함께 보존한다.
