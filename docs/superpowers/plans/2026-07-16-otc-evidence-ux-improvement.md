# OTC 근거 조회 UX 개선 구현 계획

> 실행일: 2026-07-16
> 작업 루트: `C:\dev\otc-nutrient-safety-engine`
> 제한: 커밋·푸시·배포 금지, 기존 dirty worktree 보존, 블라인드 독립평가 상태 변경 금지

## 목표

현재 식약처 허가 원문까지 검증된 13개 제품을 모두 탐색할 수 있게 하고, 결정론적 판정 결과를 미확인 조건보다 먼저 보여 준다. 각 판정에는 사용자 친화적인 식약처 출처명과 검증된 보조 학술문헌을 연결한다. 약학정보원 연결 카탈로그 369건은 연구 후보로만 집계하고 안전성 판정 제품으로 승격하지 않는다.

## 1. 데이터 경계와 표시 수치 고정

- `research_v3/otc/selection/catalog_health_kr_summary.json`의 집계만 런타임에 포함한다.
- 원본 제품명·가격·상세 약학정보는 공개 런타임에 복사하지 않는다.
- 화면에 `판매 SKU 776건 선별`, `약학정보원 연결 369건`, `식약처 허가·규칙 연결 13건`을 서로 다른 상태로 표시한다.
- `releaseReady=false`, 독립평가 미완료 상태를 유지한다.

검증:

- 런타임 빌드 테스트가 집계 수치와 `runtimePromotionAllowedCount=0`을 확인한다.
- 공개 런타임에 가격 필드가 없는지 확인한다.

## 2. 제품 탐색을 13개 전체로 확장

- `OtcProduct`에 연구 선정 약효군을 추가하고 런타임 빌더가 `class_id`에서 생성한다.
- 첫 6개로 잘라 보여 주는 코드를 제거한다.
- 약효군 필터와 전체 제품 목록을 제공한다.
- 제품명 검색은 기존처럼 식약처 검증 제품만 선택 가능하게 유지한다.
- 빠른 점검 예시를 중복 성분, NSAID 중복, 항히스타민 중복, 소화제 성분 중복, 운전 주의, 항응고제 병용으로 확장한다.

검증:

- 모든 런타임 제품이 정확히 한 약효군에 속하는지 테스트한다.
- 빠른 예시의 모든 제품 ID가 런타임에 존재하고 서로 다른 판정 유형을 실제로 생성하는지 테스트한다.

## 3. 미확인 조건을 결과 뒤로 이동하고 그룹화

- 엔진의 제품별 coverage gap 원본은 연구 추적을 위해 유지한다.
- 표시 계층에서 같은 `ruleType`의 gap을 하나로 묶고 제품명을 목록으로 보여 준다.
- 실제 finding을 먼저 표시한다.
- 미확인 조건은 중립 색상의 닫힌 `details` 안에 `추가로 확인할 조건`으로 표시한다.
- 입력 오류는 계속 첫 번째로 노출한다.

검증:

- 10개의 제품별 gap이 조건별 그룹으로 축약되는 단위 테스트를 먼저 작성한다.
- finding이 존재할 때 결과 요약이 finding 수를 우선하는지 확인한다.

## 4. 식약처 원문과 보조 학술문헌을 분리해 연결

- `research_v3/otc/rules/supporting_literature.csv`를 만든다.
- 각 행에 PMID, DOI, 논문명, 연구 유형, 연결 rule type, 한국어 핵심 결과, 적용 한계를 기록한다.
- 모든 논문은 PubMed 원문 페이지로 확인한다.
- `supports_rule_release=false`, `review_status=codex_curated_supporting_not_rule_release_evidence`를 고정한다.
- 검증 스크립트가 PMID/DOI/URL/연결 규칙/중복을 검사하고 `src/generated/otc-supporting-literature.json`을 생성한다.
- UI는 `식약처 허가사항(판정 근거)`와 `학술문헌(이해를 돕는 보조 근거)`를 구분해 표시한다.
- raw source ID 대신 `식약처 의약품안전나라 허가사항`을 표시한다.

검증:

- 연결된 모든 논문에 PMID, 제목, 핵심 결과, 한계가 있는지 테스트한다.
- 보조 문헌이 규칙 release 수나 판정 결과를 바꾸지 않는지 회귀 테스트한다.

## 5. 접근성과 반응형 UX 정리

- 검색 입력에 `name`, `type=search`, 올바른 placeholder 말줄임표를 추가한다.
- 빠른 예시·약효군 필터·제품 버튼에 명확한 선택 상태와 포커스 스타일을 제공한다.
- 모바일에서는 제품 목록과 근거 카드가 가로 넘침 없이 읽히게 한다.
- `prefers-reduced-motion`을 반영한다.

검증:

- Vitest, 연구 pytest, lint, typecheck, Next build를 실행한다.
- 로컬 개발 서버에서 데스크톱과 390px 모바일 화면을 확인한다.
- `감기약 + 해열제`와 모든 조건 입력 시 유효 판정이 먼저 나오고 미확인 조건이 접힌 상태인지 확인한다.
- 캡처 결과는 `C:\dev\otc-nutrient-safety-engine\etc\playwright\`에 둔다.

## 완료 조건

- 13개 검증 제품이 모두 탐색 가능하다.
- 776/369/13의 의미가 혼동 없이 표시된다.
- 실제 판정이 coverage gap보다 먼저 표시된다.
- coverage gap이 조건별로 그룹화되어 기본적으로 접혀 있다.
- 각 지원 rule type에 식약처 허가 원문과 검증된 보조 논문이 역할별로 표시된다.
- 보조 논문은 판정 규칙이나 완료·성능 주장에 사용되지 않는다.
- 전체 검증 명령과 브라우저 점검이 통과한다.
