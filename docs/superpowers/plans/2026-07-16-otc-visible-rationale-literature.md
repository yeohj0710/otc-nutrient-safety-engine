# OTC 판정 논거·논문 직접 표시 구현 계획

> 실행일: 2026-07-16
> 작업 루트: `C:\dev\otc-nutrient-safety-engine`
> 제한: 커밋·푸시·배포 금지, 기존 dirty worktree 보존, 블라인드 독립평가 상태 변경 금지

## 목표

각 안전성 판정 카드가 상담 권고만 보여 주지 않고, 판정을 촉발한 입력·제품·성분, 식약처 허가 원문, 직접 연결한 학술문헌의 결과와 한계를 접힌 영역 밖에서 설명하도록 개선한다. 허가 원문은 결정론적 판정 근거로 유지하고, 학술문헌은 판정의 이해를 돕는 보조 근거로만 표시한다.

## 1. 현재 근거 손실을 회귀 테스트로 고정

- `rule_evidence_shortlist.csv`의 허가 원문이 `build_runtime.py`에서 사라지는 경로를 테스트한다.
- 런타임의 규칙 근거에 규칙 ID, 제품명, 한국어 원문 인용문이 반드시 포함되는지 확인한다.
- 잘린 문장 조각이 사용자 화면에 전달되지 않도록 대표 규칙의 완결된 인용문을 검증한다.

검증:

- 새 테스트가 현재 구현에서 실패하는지 먼저 확인한다.
- 제품·규칙·출처 locator 연결 수는 기존 수치를 유지한다.

## 2. 허가 원문과 판정 조건을 런타임에 보존

- 런타임 규칙 근거에 `ruleId`, `productName`, `excerptKo`를 추가한다.
- 술, 임신, 항응고제, 진정 작용 약물, 고혈압 관련 잘린 인용문을 추출 원문과 대조해 완결된 문장으로 고친다.
- 규칙 유형별 관련 성분을 우선해 판정 이유에 표시하고, 연결 정보가 없을 때만 제품의 전체 성분을 사용한다.

검증:

- 생성된 `otc-runtime.json`에 모든 released 규칙의 원문 인용문이 들어 있는지 확인한다.
- `complete`, `releaseReady`, `performanceClaimAllowed`, 독립평가 상태가 바뀌지 않았는지 확인한다.

## 3. 부족한 규칙별 학술문헌을 보강

- PubMed에서 음주·아세트아미노펜, 만성 간질환·아세트아미노펜, 경구 비충혈제거제·혈압, 임신 후기 NSAID 위험을 우선 조사한다.
- PMID, DOI, 연구설계, 한국어 핵심 결과, 규칙 연결 이유, 적용 한계를 기록한다.
- 연구 결과가 허가 경고를 약화하거나 과장하지 않도록 `주의 지지`, `불확실성 설명`, `기전 보조` 역할을 구분한다.
- 직접 맞는 논문이 없는 규칙은 억지로 연결하지 않고 ‘직접 연결 문헌 없음’을 명시한다.

검증:

- PMID·DOI·중복·규칙 유형·필수 설명 필드를 테스트한다.
- 보조 문헌 추가가 규칙 release 상태나 엔진 판정을 바꾸지 않는지 확인한다.

## 4. 결과 카드의 정보 순서를 변경

- 카드 첫 화면에 `판정 이유`를 표시한다: 입력 조건, 제품명, 관련 성분·함량.
- 바로 아래에 `식약처 허가 원문` 인용문과 원문 locator 링크를 표시한다.
- 직접 연결 논문이 있으면 첫 논문의 핵심 결과, 연결 이유, 한계, 논문명·PMID 링크를 표시한다.
- 직접 연결 논문이 없으면 허가 원문만 사용한 판정임을 명시한다.
- `지금 할 일`은 논거 뒤에 배치한다. 전체 근거 목록은 기존 상세 펼침 영역에 유지한다.

검증:

- 카드가 접힌 상태에서도 판정 이유·허가 인용문·논문 미리보기 또는 문헌 없음 상태를 볼 수 있는지 테스트한다.
- 논문 연결 수와 허가 원문 연결 수를 결과 요약에 표시한다.

## 5. 전체 회귀·화면 검증

- 연구 pytest, 앱 Vitest, lint, typecheck, Next build를 실행한다.
- 데스크톱과 390px 모바일에서 여러 질환·음주·임신 조건을 함께 선택한 사례를 확인한다.
- 근거 카드의 한국어 문장, 긴 논문명, 출처 링크가 넘치거나 가려지지 않는지 확인한다.
- 결과 캡처는 `C:\dev\otc-nutrient-safety-engine\etc\playwright\`에 둔다.

## 완료 조건

- 상담 권고 전에 구체적인 판정 이유와 근거가 보인다.
- 모든 released 규칙은 식약처 허가 원문 인용문을 런타임에 보존한다.
- 직접 맞는 논문은 결과·연결 이유·한계와 함께 보이고, 없는 논문은 없다고 밝힌다.
- 보조 논문은 결정론적 판정과 규칙 승인 근거로 오인되지 않는다.
- 전체 테스트와 빌드가 통과하고 연구 완료·성능 상태는 기존 경계를 유지한다.
